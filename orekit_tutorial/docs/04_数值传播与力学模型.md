# 第04章：轨道传播与力学模型

本章学习如何使用传播器计算轨道，并对比不同力学模型的组合效果。

## 对应代码文件

| 文件 | 说明 |
|------|------|
| `04_keplerian_propagation.py` | 开普勒传播（两体问题），传播原理、周期验证、轨道闭合性验证 |
| `04_numerical_propagation.py` | 数值传播与力学模型对比，包含多种摄动组合的详细对比 |

## 4.1 什么是轨道传播

轨道传播（Propagation）是根据当前轨道状态，计算未来或过去某个时刻的轨道状态。

```
初始状态 (t0) --> 传播器 --> 目标状态 (t1)
```

Orekit 提供两种主要的传播方法：

| 方法 | 类 | 特点 |
|------|-----|------|
| 解析法 | `KeplerianPropagator` | 快速，仅考虑中心引力 |
| 数值法 | `NumericalPropagator` | 精确，可加入多种力学模型 |

## 4.2 开普勒传播（两体问题）

最简单的传播：只考虑地球中心引力，不考虑摄动。

```python
from orekit_init import *

date = create_date(2024, 1, 1, 12, 0, 0)
mu = Constants.EGM96_EARTH_MU
gcrf = FramesFactory.getGCRF()

# 创建初始轨道
orbit = KeplerianOrbit(
    7_000_000.0, 0.001,
    math.radians(98.0), math.radians(90.0), 0.0, 0.0,
    PositionAngle.TRUE, gcrf, date, mu
)

# 创建开普勒传播器
propagator = KeplerianPropagator(orbit)

# 传播 45 分钟
target_date = date.shiftedBy(2700.0)
final_state = propagator.propagate(target_date)
print_orbit(final_state.getOrbit(), "传播 45 分钟后")
```

## 4.3 数值传播（考虑摄动）

使用数值积分器，可以加入各种力学模型：

```python
from org.hipparchus.ode.nonstiff import DormandPrince853Integrator

# 创建数值传播器
integrator = DormandPrince853Integrator(
    1.0,        # 最小步长 (s)
    600.0,      # 最大步长 (s)
    1.0e-9,     # 绝对精度
    1.0e-12     # 相对精度
)
propagator = NumericalPropagator(integrator)
propagator.setOrbitType(OrbitType.EQUINOCTIAL)

# 设置初始状态
initial_state = SpacecraftState(orbit, 100.0)
propagator.setInitialState(initial_state)

# 添加中心引力
propagator.addForceModel(NewtonianAttraction(mu))
```

## 4.4 添加力学模型

### 地球重力场（6x6 阶）

```python
from org.orekit.forces.gravity import HolmesFeatherstoneAttractionModel
from org.orekit.forces.gravity.potential import GravityFieldFactory

itrf = FramesFactory.getITRF(IERSConventions.IERS_2010, True)
provider = GravityFieldFactory.getNormalizedProvider(6, 6)
propagator.addForceModel(HolmesFeatherstoneAttractionModel(itrf, provider))
```

### 大气阻力

```python
from org.orekit.models.earth.atmosphere import HarrisPriester
from org.orekit.forces.drag import DragForce, IsotropicDrag
from org.orekit.bodies import OneAxisEllipsoid

earth = OneAxisEllipsoid(
    Constants.WGS84_EARTH_EQUATORIAL_RADIUS,
    Constants.WGS84_EARTH_FLATTENING, itrf
)
sun = CelestialBodyFactory.getSun()
atmosphere = HarrisPriester(sun, earth)
sc_drag = IsotropicDrag(0.01, 2.2)
propagator.addForceModel(DragForce(atmosphere, sc_drag))
```

### 日月引力（第三体引力）

```python
from org.orekit.forces.gravity import ThirdBodyAttraction

sun = CelestialBodyFactory.getSun()
moon = CelestialBodyFactory.getMoon()
propagator.addForceModel(ThirdBodyAttraction(sun))
propagator.addForceModel(ThirdBodyAttraction(moon))
```

### 太阳光压

```python
from org.orekit.forces.radiation import SolarRadiationPressure, IsotropicRadiationSingleCoefficient

sc_srp = IsotropicRadiationSingleCoefficient(0.01, 1.5)
earth_radius = Constants.WGS84_EARTH_EQUATORIAL_RADIUS
propagator.addForceModel(SolarRadiationPressure(sun, earth_radius, sc_srp))
```

## 4.5 传播精度控制

数值传播器可以控制积分精度：

```python
integrator = DormandPrince853Integrator(
    min_step,    # 最小步长 (s)
    max_step,    # 最大步长 (s)
    abs_accuracy,# 绝对误差容限 (m)
    rel_accuracy # 相对误差容限 (无量纲)
)
```

- 步长越小，精度越高，但计算时间越长
- 通常 abs_accuracy 设为 1e-9，rel_accuracy 设为 1e-12

## 4.6 三种场景对比

`04_numerical_propagation.py` 中对比了三种力学模型组合：

| 场景 | 力学模型 | 说明 |
|------|---------|------|
| 场景1 | 仅中心引力 | 理想两体问题 |
| 场景2 | 中心引力 + 重力场 | 包含地球非球形摄动 |
| 场景3 | 中心引力 + 重力场 + 日月引力 | 最完整的力学模型 |
